# Skill: B2B Design Criteria

## Purpose
레퍼런스 사이트 3개와의 직접 비교 기준을 제공한다.
"구현 여부"가 아닌 "수준 비교"가 핵심이다.

## Palantir 기준 (100점)
직접 확인한 원칙:
- 헤드라인: 짧고 도발적, 설명 없이 선언
- 배경: 다크 네이비 — 신뢰감과 무게감
- 데이터: 숫자가 카피보다 먼저 눈에 들어옴
- 비교: "우리가 다른 이유"를 방어 없이 증명
- AI 패턴: 전혀 없음 — 독자적 디자인 언어

## Veeva 기준 (페르소나 구조)
- 페르소나 선택이 Hero 영역 안에서 해결됨
- 각 역할별 "내가 할 수 있는 것"을 동사로 표현
- 모듈 카드: 독립 가치 + 전체 흐름 동시 표현

## S&P Global MI 기준 (데이터 플랫폼 신뢰)
- 복잡한 멀티 솔루션을 카테고리 레이어로 정리
- 숫자 지표: Hero 바로 아래, 크고 굵게
- Supply Chain 도메인: 구체적 카테고리 명시

## 감점 패턴 — 즉시 감점
- 흰 카드 + 연보라/연파랑 그라디언트 배경
- 섹션마다 이모지 남발 (🧠🔍⚖️⚡)
- "Unlock the power of..." 류의 카피
- 모든 섹션 동일한 패딩/여백
- 아이콘만 있고 실제 데이터 없는 카드
- 그림자 과다 사용

## 가점 패턴 — Palantir 수준
- 실제 숫자 데이터가 디자인 요소로 사용됨
- 고객 로고 실명 (또는 명확한 TBU)
- 비교표에 구체적 경쟁사 약점 명시
- CTA가 "다음에 무슨 일이 일어나는지" 설명
- 섹션마다 시각적 변화 (배경색, 레이아웃 교차)

## CTA 기준
좋은 예:
- "Get My Tailored Demo →"
- "See Tridge Eye in Action →"
- "Request Executive Demo →"

나쁜 예:
- "Get Started"
- "Learn More"
- "Contact Us"

## 카피라이팅 금지
- "Without a Sales Team" → "At a Scale Your Team Can't Do Alone"
- "Zero Ops Burden" → "Your team, amplified"
- 통계 오류: 750K+, 195+ 사용 금지
